To introduce (or edit) a node whose statement includes any mathematical difference from the corresponding (explicit or implicit) statement in the reference paper (e.g., a changed constant, or changed hypotheses/conclusion of a helper or lemma) requires an authorization for a deviation. This should usually be avoided in favor of simply following the exact paper path completely faithfully.

To write the node with a requested deviation, write a TeX-only file under `reference/`, add a `deviation_requests` entry, and claim the deviation in `node_deviation_claims` for each affected node. Use `node_deviation_claims` only for authorized deviations or deviations requested in this response. The deviation will be checked against the DEVIATIONS.md rubric.

To retire a deviation that is no longer needed: (1) `rm` the `reference/<path>.tex` file on disk, (2) emit a `node_deviation_claims` entry for each node that previously claimed it, dropping the id from that node's claim set (or empty the set if it was the only claim), and (3) list the deviation id in `deviation_deletions`. The kernel rejects the response if any node still claims a to-delete id after these updates are applied, or if the file still exists on disk; clear every claim AND remove the file in the same response (or run a prior burst that clears them first).
